Test Automation Framework : Things to consider to enable RV to reuse your test cases

This page last changed on Feb 07, 2013 by qshatus.

  • Make sure there are no local dependencies in your test project

    • E.g. using a local jboss, hard coding a local sever

  • Host details (the use of & the format)

    • Remember RV will want to run your test cases on their environment so they will need to replace your host details with the details of the host in the RV setup

    • Use the TAF standard for the format of the host properties file

      • Host Properties File Structure and Example

    • Always use a host object or it's properties

  • Test cases need to be runnable on Jenkins

    • Ideally from running the “mvn clean install” command
